1. 背景与目标
当前 FinancialReportModal 已有「资产负债快照（BALANCE）」和「流水日志（JOURNAL）」两个标签页，但缺少一张能清晰呈现每次买卖/转换/借款操作明细的结构化表格，用户无法快速定位某次操作的执行价、股数、损益，也无法将数据导出做二次分析。
ResultsDashboard 的净值曲线（Recharts LineChart）目前也缺少操作点位标注，用户无法直观确认策略在哪个时点执行了什么操作。

本次迭代目标：
在 FinancialReportModal 新增「操作记录（OPERATION_LOG）」标签页，呈现类似附图Excel表格的逐行操作明细，并支持一键导出CSV。
在 ResultsDashboard 净值曲线图上叠加操作标注点（ReferenceDot），鼠标悬停时展示详细操作数据。
补充用户需求文档未涵盖但实现完整功能必需的若干细节（见第6节）。

表格参照截图，包含以下列，按操作发生的时间顺序排列（最新在前）：

列名	来源字段	说明	类型

质借累积	debtBalance	质押借款累积债务余额（BEAR_PLEDGE 策略专属）	number$
质押库存	pledgeBatches 合计市值	当前仍持有的质押买入仓位合计市值	number$

2.4 分组与筛选
默认按「操作阶段」分组显示，对应截图中的「0.初始状态」「1.下跌加码」「2.回到高点」「3.上涨减码」。
分组判断逻辑：从 strategyMemory.bearPhase / recoveryPhase 读取，bearPhase > 0 → "下跌加码"，recoveryPhase > 0 → "上涨减码"，两者均为 0 且 totalValue 低于 HWM → "回到高点"，其余 → 正常运行。
工具栏提供以下筛选控件：
类型下拉：全部 / 买入 / 卖出 / 借款 / 还款 / 系统事件
时间范围：年份下拉（从回测起始年到结束年），选择后只显示该年数据
策略阶段：全部 / 下跌加码 / 上涨减码 / 正常运行
筛选后「汇总行」在表格顶部实时更新，显示当前视图的操作次数、总买入额、总卖出额、净额。

2.5 汇总行（Summary Row）
表格顶部固定一行汇总（类似Excel冻结行），样式为蓝色背景：
汇总列	计算逻辑

总买入金额	所有 amount > 0 的操作金额合计
总卖出/还款金额	所有 amount < 0 的操作金额合计（绝对值）
净买入	买入 - 卖出，正值表示净流入股市
期末总资产	最后一条记录的 totalAssets
期末损益	最后一条记录的 totalValue - initialCapital

该函数内部调用 buildOperationLogRows（见 §2.7）构建数据行，再生成 Blob 并触发下载。

2.7 数据构建函数
新增文件 services/operationLogService.ts，核心导出函数：
export interface OperationLogRow {
  date: string
  status: string          // 原始 event.type
  statusCN: string        // 中文映射
  qqqPrice: number
  changePct: number       // QQQ 当月涨跌%
  phasePct: number        // 阶段触发阈值%（如 -10, -20 ...）
  sharesChanged: number | null
  amount: number
  sharesAfter: number
  stockValue: number
  totalAssets: number
  pnl: number
  pnlPct: number
  maintenanceRatioPct: number | null
  pledgeCumulative: number
  pledgeInventoryValue: number
  groupLabel: string      // 分组标签：正常/下跌加码/上涨减码/回到高点
}

export function buildOperationLogRows(
  result: SimulationResult,
  marketData: MarketDataRow[]
): OperationLogRow[]

实现要点：
遍历 result.history，对每个 PortfolioState 中的每个 FinancialEvent 生成一行。
没有 event 的月份（仅持有无操作）不生成行（降低噪音）——除非是初始状态月份。
涨跌% 通过与上一个 PortfolioState 的 qqqClose 对比计算（需同时传入 marketData）。
phasePct 从 strategyMemory.bearPhase / recoveryPhase 映射：bearPhase=1→-10，=2→-20，=3→-30，=4→-40；recoveryPhase=1→+10，...=4→+40；其余→0。
sharesChanged：读取 TRADE/ROTATION/PLEDGE 事件中涉及标的的股数差值（amount / price）。其他类型为 null。
pledgeInventoryValue：遍历 strategyMemory.pledgeBatches，将每批 qty × currentPrice 累加。


3. 功能二：净值曲线折线图操作点标注
3.1 目标效果
在 ResultsDashboard 的主净值曲线图（Recharts ComposedChart / LineChart）上，对每个有操作的月份叠加彩色标注点，用户鼠标悬停时显示一个弹窗，展示该月所有操作的详细信息，便于视觉验证策略逻辑。

3.2 标注点（ReferenceDot）规格
使用 Recharts <ReferenceDot> 组件，每个有操作的月份渲染一个标注点：
操作类型	dot 颜色	dot 大小 r	形状（ifOverflow）
ROTATION_IN（买入3×）	#f59e0b (amber)	5	extendDomain
ROTATION_OUT（卖出3×）	#10b981 (green)	5	extendDomain
PLEDGE_BORROW（质押买）	#f97316 (orange)	5	extendDomain
PLEDGE_REPAY（质押还）	#3b82f6 (blue)	5	extendDomain
TRADE Buy（定投）	#6366f1 (indigo)	3	extendDomain
TRADE Sell（卖出）	#ef4444 (red)	3	extendDomain
WITHDRAW（提款）	#8b5cf6 (violet)	4	extendDomain
INFO/ALERT（爆仓）	#dc2626 (red-600)	7	extendDomain

??  同一月份若存在多个不同类型操作，优先展示优先级最高的一个点（优先级：INFO > ROTATION > PLEDGE > TRADE > DEPOSIT > WITHDRAW）。当月所有操作均在 Hover 弹窗中完整展示。

3.3 Hover 弹窗（Custom Tooltip 扩展）
3.3.1 触发方式
扩展现有 CustomTooltip 组件。当 active=true 且 label 对应日期存在操作记录时，在原始净值信息下方渲染操作明细区域。

3.3.2 弹窗内容结构
区域	内容
顶部（现有）	日期 + 各策略净值（原有逻辑保持不变）
分隔线	仅当有操作时显示
操作明细标题	「本月操作 (N 条)」
操作行（循环）	每条 FinancialEvent → [类型badge] [描述] [金额]
底部数据	QQQ 当月涨跌% · 当前 bearPhase / recoveryPhase（仅熊市策略）

3.3.3 弹窗样式要求
最大宽度 340px，有操作时自动展开；背景白色，圆角 12px，阴影 shadow-xl。
操作类型 badge 使用和标注点一致的颜色体系（§3.2 表格中的颜色）。
金额正值绿色显示，负值红色显示。
弹窗不超出图表边界（Recharts tooltip position 使用 `wrapperStyle.zIndex: 9999`）。

3.4 图表图例扩展
在净值曲线图例（Legend）下方新增标注点图例区域：
水平排列 5 个彩色圆点 + 文字：买入 · 卖出 · 转入3× · 转出3× · 质押 · 系统事件。
